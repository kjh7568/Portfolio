Hive Survivor / Feature 01 · Hex Grid Pathfinding
01 FEATURE · HEX GRID PATHFINDING

육각 그리드 위에서
100마리가 동시에 움직이게 하기.

셀 단위 A*에 계층화 캐싱을 더해, 적 100유닛 동시 경로 계산을 16ms 이내로 안정화했습니다.

구현 배경
Why this feature?

초기 빌드는 사각 그리드 + Vector3 기반 직선 추적이었으나, 적이 30유닛을 넘는 순간 추적 알고리즘이 메인 스레드에서 30ms 이상을 점유했습니다. 또한 사각 그리드의 8방향 이동은 벌집 컨셉과 시각적으로 어긋났습니다.

해결 방향: (1) 게임 셀을 axial 좌표의 육각 그리드로 교체, (2) 적 그룹을 클러스터로 묶어 같은 도착지에 대해 경로를 1회만 계산, (3) 매 프레임이 아닌 200ms 간격으로 다시 계산하고 그 사이는 path follow만 수행.

설계 의도

적 AI가 매 프레임 길찾기를 호출하지 않도록, 요청을 큐에 모아 비동기로 처리합니다. 순수 C# 레이어와 MonoBehaviour 레이어를 분리해 테스트 가능성을 확보했습니다.

핵심 클래스 3개

HexCoord: axial(q,r) 구조체. 거리/이웃 계산을 매 프레임 반복하므로 struct + IEquatable로 GC를 차단했습니다.

GridService: 셀 가중치(이동비용)와 차단 상태를 보관. 갱신은 이벤트로 broadcast.

PathCache: 같은 (start, goal)에 대한 경로를 200ms TTL로 캐싱. 같은 도착지에 모이는 적 클러스터에서 절대적인 효과.

DI

세 클래스 모두 인터페이스로 추상화 후 Zenject로 주입합니다. 테스트에서는 fake grid를 주입해 5×5 같은 미니맵으로 단위 검증.

Axial 좌표계와 캐싱 전략

1 · 좌표계 선택 — Axial

DECISION

육각 그리드는 offset · cube · axial 표현이 가능합니다. 메모리(2개 정수)와 거리 계산 단순성의 균형을 위해 axial(q, r)을 채택. 거리 함수는 (|q| + |q+r| + |r|) / 2로 분기 없이 계산됩니다.

2 · 이웃 룩업 테이블

PERF

이웃 6방향 오프셋을 정적 배열로 보관해 매 호출 GC alloc을 0으로 유지했습니다. readonly static field로 노출.

22public readonly struct HexAStar
{
    private readonly IGridService _grid;
    private readonly PathCache    _cache;

    public PathResult FindPath(HexCoord from, HexCoord goal)
    {
        if (_cache.TryGet(from, goal, out var cached)) return cached;

        var open  = PriorityQueuePool<HexCoord>.Rent();
        var came  = DictPool<HexCoord, HexCoord>.Rent();

        open.Enqueue(from, 0);
        while (open.Count > 0)
        {
            var cur = open.Dequeue();
            if (cur.Equals(goal)) break;
            // 6 neighbors lookup — zero alloc
            foreach (var n in _grid.Neighbors(cur)) { /* relax edge */ }
        }
        return _cache.Store(from, goal, Reconstruct(came, goal));
    }
}

Hive Survivor / Feature 01 / Troubleshooting
TROUBLESHOOTING — TS-01

Wave 4 진입 시 16ms 스파이크

90마리 적이 동시에 같은 플레이어를 향해 경로 재계산을 요청하면서 메인 스레드 1프레임을 통째로 점유.

01SYMPTOM

Wave 4 진입 직후 0.5초간 16~22ms 스파이크가 매 프레임 발생. Profiler 상 HexAStar.FindPath가 한 프레임 내에 90회 호출됨.

02ROOT CAUSE

적 AI가 FixedUpdate마다 길찾기를 호출하도록 작성되어 있었고, 적 풀이 한꺼번에 활성화되면서 같은 프레임에 모두 1차 경로를 요구. 캐시는 비어 있는 상태였음.

03FIX

(a) 경로 요청을 큐로 받고, 한 프레임 당 최대 16개만 처리하도록 throttle. (b) 적 활성화 시점을 8프레임에 걸쳐 분산. (c) 적 spawn 시 부모(보스)의 경로를 미리 계산해 캐시 워밍.

12public async UniTask RunAsync(CancellationToken ct)
{
    var elapsed = 0f;
    while (!ct.IsCancellationRequested)
    {
        var intensity = _curve.Sample(elapsed);                    // 0..1
        var count     = Mathf.CeilToInt(intensity * _maxPerTick);
        for (var i = 0; i < count; i++)
            _spawner.Spawn(_catalog.PickWeighted(intensity));
        await UniTask.Delay(_tickMs, cancellationToken: ct);
        elapsed += _tickMs * 0.001f;
    }
}

03 FEATURE · OBJECT POOL · DAMAGE

투사체 1500개에도
GC alloc 0을 유지합니다.

투사체와 피격 데미지 구조체를 제너릭 풀로 묶어, 60FPS 가동 중 GC.Alloc 호출을 0회로 유지했습니다.

구현 배경
Why this feature?

서바이버 장르 특성상 후반부 투사체 수가 1000개를 넘기며, 매 발사마다 new + Destroy 패턴은 GC 스파이크의 직접 원인이었습니다.

풀은 단순한 큐가 아니라 “대여 → 사용 → 반납” 약속을 강제하는 인터페이스(IPoolable)로 설계해, 누가 반납을 잊더라도 5초 후 자동 회수되는 안전망을 두었습니다.

DESIGN
IPoolable + Generic Pool<T>
DAMAGE
readonly struct DamagePacket — heap 회피
RESULT
GC.Alloc 0B / 30s · 메모리 −42%

Hive Survivor / Feature 03 / Troubleshooting
TROUBLESHOOTING — TS-03

투사체가 화면 밖에서 사라지지 않는 누수

Pool에서 빌려간 투사체가 충돌·소멸 처리 분기 누락으로 5분 뒤 메모리 1.2GB까지 상승.

01SYMPTOM

플레이 5분 경과 시 메모리 사용량이 선형으로 증가, Pool의 RentedCount가 0으로 돌아오지 않음.

02ROOT CAUSE

화면 밖으로 나간 투사체가 OnBecameInvisible 콜백을 받지 못하는 카메라 설정에서 반납 분기로 들어가지 않았음. 수동 검사도 누락.

03FIX

(a) Pool에 TTL(기본 5초)을 부여해 시한 자동 회수. (b) Bounds 기반 컬링으로 수동 체크. (c) 에디터 전용 PoolInspector를 만들어 미반납 인스턴스를 시각화.

04RESULT
RESULT

30분 플레이 누수 1.2GB → 0KB. Pool 미반납으로 인한 회귀 버그를 잡기 위해 에디터 인스펙터를 신규 도구로 남겼고, 다른 시스템에서도 재사용 중입니다.

11public IEnumerable<Buff> QueryActiveBuffs(ReadOnlySpan<SkillId> owned)
{
    foreach (var id in owned)
    {
        if (!_edges.TryGetValue(id, out var outs)) continue;
        foreach (var edge in outs)
            if (edge.RequirementsMet(owned))
                yield return edge.Buff;
    }
